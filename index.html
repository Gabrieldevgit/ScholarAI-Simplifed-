🛠️ File Upload Backend — Full Setup Guide

Right now the file upload button exists but does nothing because browsers cannot call AI APIs with raw files — the file needs to be read, converted, and sent server-side. This guide walks you through creating a tiny Node.js backend that bridges the gap.

📋 What you will need

  • Node.js 18+ — nodejs.org
  • A terminal (Command Prompt, PowerShell, or macOS/Linux terminal)
  • Your Groq / OpenRouter / Hugging Face / Cohere API key
  • ~10 minutes

🗂️ Step 1 — Create your project folder

1

Open your terminal and run these commands one by one:

# Create a new folder for the backend
mkdir scholarai-backend
cd scholarai-backend

# Initialise a Node project (accept all defaults)
npm init -y

# Install the required packages
npm install express cors multer groq-sdk dotenv

express = web server  |  cors = allow browser requests  |  multer = handle file uploads  |  groq-sdk = Groq AI client  |  dotenv = load your API key safely

🔑 Step 2 — Create the .env file

2

Inside scholarai-backend/ create a file called exactly .env (no other extension) and paste:

GROQ_API_KEY=gsk_your_actual_key_here
PORT=3001

⚠️ Never commit this file to Git. Add .env to your .gitignore.

📄 Step 3 — Create server.js

3

Create a file called server.js in the same folder with this content:

import express from 'express';
import cors    from 'cors';
import multer  from 'multer';
import Groq    from 'groq-sdk';
import 'dotenv/config';
import fs      from 'fs';
import path    from 'path';

const app    = express();
const groq   = new Groq({ apiKey: process.env.GROQ_API_KEY });
const upload = multer({ dest: 'uploads/' });

app.use(cors());           // Allow requests from your HTML file
app.use(express.json());   // Parse JSON bodies

// ── Route 1: plain text chat (no file) ───────────────────────
app.post('/api/chat', async (req, res) => {
  const { messages, model, systemPrompt } = req.body;
  try {
    const response = await groq.chat.completions.create({
      model:    model || 'llama-3.3-70b-versatile',
      messages: [{ role: 'system', content: systemPrompt }, ...messages],
      max_tokens: 4096
    });
    res.json(response);
  } catch(err) {
    res.status(500).json({ error: err.message });
  }
});

// ── Route 2: chat WITH a file attachment ─────────────────────
app.post('/api/chat-with-file', upload.single('file'), async (req, res) => {
  const { messages, model, systemPrompt } = req.body;
  const file = req.file;

  let fileContent = '';
  if (file) {
    const ext = path.extname(file.originalname).toLowerCase();
    if (['.txt', '.md', '.js', '.py', '.html', '.css', '.json'].includes(ext)) {
      // Text files: read directly
      fileContent = fs.readFileSync(file.path, 'utf8');
    } else {
      fileContent = `[Binary file: ${file.originalname} (${file.mimetype})]`;
    }
    fs.unlinkSync(file.path); // Delete temp file after reading
  }

  const parsedMessages = JSON.parse(messages);
  if (fileContent) {
    parsedMessages[parsedMessages.length - 1].content +=
      `\n\n---\nFile contents of "${file.originalname}":\n\`\`\`\n${fileContent}\n\`\`\``;
  }

  try {
    const response = await groq.chat.completions.create({
      model:    model || 'llama-3.3-70b-versatile',
      messages: [{ role: 'system', content: systemPrompt }, ...parsedMessages],
      max_tokens: 4096
    });
    res.json(response);
  } catch(err) {
    res.status(500).json({ error: err.message });
  }
});

app.listen(process.env.PORT || 3001, () =>
  console.log(`✅ ScholarAI backend running on port ${process.env.PORT || 3001}`)
);

📦 Step 4 — Fix package.json for ES Modules

4

Open package.json and add "type": "module" so the import statements work:

{
  "name": "scholarai-backend",
  "type": "module",
  "scripts": {
    "start": "node server.js"
  }
}

▶️ Step 5 — Start the backend

5

In your terminal (inside scholarai-backend/) run:

node server.js

You should see: ✅ ScholarAI backend running on port 3001

Leave this terminal open while you use ScholarAI.

🔌 Step 6 — Connect ScholarAI.html to your backend

6

In ScholarAI.html, find the sendMessage function and replace the fetch call with:

// Replace the fetch inside sendMessage with this:
const res = await fetch('http://localhost:3001/api/chat', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    messages: state.messages,
    model:    state.selectedModel,
    systemPrompt: SYSTEM_PROMPTS[state.selectedMode]
  })
});

And for file upload, change the handleFileUpload function to send a FormData to /api/chat-with-file instead of showing an alert.

🚀 Deployment (optional — make it live online)

Once everything works locally you can host the backend for free on Render.com or Railway.app:

  • Push your scholarai-backend folder to a GitHub repo (without the .env file)
  • On Render/Railway, connect the repo and add your GROQ_API_KEY as an Environment Variable in their dashboard
  • Both platforms auto-detect Node and run npm start for you
  • Update the fetch URL in ScholarAI.html from localhost:3001 to your live URL

📁 Final folder structure

scholarai-backend/
├── server.js        ← Your backend code
├── package.json     ← "type": "module" added
├── .env             ← Your API key (never commit!)
├── .gitignore       ← Contains: node_modules/ and .env
└── node_modules/    ← Auto-created by npm install
