🛠️ Backend Setup — Simple Guide

You need a backend for two things: file uploads and using Together AI / DeepSeek (those APIs block direct browser requests). The simplest fix uses only Python — no Node.js, no npm, nothing to install.

✅ Option A — Serve the file (fixes Together AI & DeepSeek, 10 seconds)

Open a terminal in the same folder as ScholarAI.html and run:

python -m http.server 8080

Then open http://localhost:8080/ScholarAI.html — all providers work. (Click the box to copy the command)

📁 Option B — Full backend with file upload support

Requires Python 3 (already on Mac/Linux; download for Windows). No other installs needed.

1

Create a file called backend.py in the same folder as ScholarAI.html and paste this:

import http.server, json, urllib.request, urllib.error, os, ssl, cgi

PORT       = 8080
UPLOAD_DIR = "uploads"
if not os.path.exists(UPLOAD_DIR): os.makedirs(UPLOAD_DIR)

class Handler(http.server.SimpleHTTPRequestHandler):
    def do_OPTIONS(self):
        self.send_response(200); self._cors(); self.end_headers()

    def do_POST(self):
        if self.path == '/proxy':
            length = int(self.headers['Content-Length'])
            data   = json.loads(self.rfile.read(length))
            url    = data.pop('__url')
            key    = data.pop('__key')
            xhdrs  = data.pop('__headers', {})
            req    = urllib.request.Request(url,
                data=json.dumps(data).encode(),
                headers={'Content-Type':'application/json',
                         'Authorization':f'Bearer {key}', **xhdrs},
                method='POST')
            ctx = ssl.create_default_context()
            try:
                with urllib.request.urlopen(req, context=ctx) as r:
                    body = r.read()
                self.send_response(200)
            except urllib.error.HTTPError as e:
                body = e.read(); self.send_response(e.code)
            self._cors(); self.send_header('Content-Type','application/json')
            self.end_headers(); self.wfile.write(body)

        elif self.path == '/upload':
            ctype, pdict = cgi.parse_header(self.headers['content-type'])
            if ctype == 'multipart/form-data':
                pdict['boundary'] = pdict['boundary'].encode()
                fs       = cgi.FieldStorage(fp=self.rfile, headers=self.headers,
                               environ={'REQUEST_METHOD':'POST'})
                fileitem = fs['file']
                filename = os.path.basename(fileitem.filename)
                dest     = os.path.join(UPLOAD_DIR, filename)
                with open(dest, 'wb') as f:
                    f.write(fileitem.file.read())
                result = json.dumps({"status":"ok","filename":filename}).encode()
                self.send_response(200); self._cors()
                self.send_header('Content-Type','application/json')
                self.end_headers(); self.wfile.write(result)
            else:
                self.send_response(400); self.end_headers()
        else:
            self.send_response(404); self.end_headers()

    def _cors(self):
        self.send_header('Access-Control-Allow-Origin','*')
        self.send_header('Access-Control-Allow-Headers','*')
        self.send_header('Access-Control-Allow-Methods','POST, OPTIONS')
    def log_message(self,*a): pass

print(f"✅ Backend Active: http://localhost:{PORT}/ScholarAI.html")
http.server.HTTPServer(('',PORT),Handler).serve_forever()
2

Open a terminal in that folder and run:

python backend.py

You'll see: ✅ Open → http://localhost:8080/ScholarAI.html

Open that URL. All 5 providers work and file upload will be enabled. Leave the terminal open while using ScholarAI. (Click code to copy)

3

That's it. No npm, no installs, no config files. If you see "python not found" try python3 backend.py. On Windows you can also double-click the backend.py file.
